Сделать REST фото менеджер.

1) загружать фотографии авторизованным пользователям.
    1.1) при загрузке можно указывать различную метадату: гео локацию, описание, имена людей на фото.
2) отображать список фотографий, без мета данных.
    2.2) фильтровать фотографии по дате.
    2.3) фильтровать фотографии по геолокации.
    2.4) фильтровать фотографии по имени человека.
3) получать фотографию по айди с метаданными.
4) доп задача: сделать апи автодополнение по поиску возможных имен людей присутствующих на фотографиях.
Пример:
передаем часть имени “Алекс”
на выходе получаем:
Алекс
Алексей
Александр
Александра
Вбиваем Алексан:
получаем
Александр
Александра

Tech stack:
Django
Python 3.9+
Any SQL database
DRF is optional

В решении используется БД SQLight. Фильтрация по именам людей на фотографии, поиск с автозаполнением работают лотько на
латинице. Для хранения метаданных исполюзуется json-field, БД плохо работает с кириллицей в json-fiel'ах.
Python 3.10
localhost:port/docs/ - документация
Фильтры:
1. По активности;
2. По дате:
    2.1 Точное указание даты в формате дд.мм.гггг или гггг-мм-дд. Тип данных string
    2.2 Дата до в формате дд.мм.гггг или гггг-мм-дд. Тип данных string
    2.2 Дата после в формате дд.мм.гггг или гггг-мм-дд. Тип данных string
3. Фильр по имени (только латиница). Тип данных string
4. Фильтр по геолокации. Формат входных данных string. Строка содержит точку геолокации (lat, lon) в формате
01.0123456,01.0123456. При необходимости, через запятую можно указать радиус поиска в километрах
01.0123456,01.0123456,30. Если радиус указан, при формировании ответа между заданной точкой и точкой локации на фото
вычисляется геодезическое расстояние.
5. Доп задача по автозаполнению формы:
После формы фильтров добавлена дополнительная форма поиска с автозаполнением (только латиница).
